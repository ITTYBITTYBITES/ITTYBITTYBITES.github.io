## Purpose

This document defines **how** we build the platform.

It will cover:

- Code organization and module boundaries
- Experience module contract (must remain stable)
- Registry and discovery rules
- Performance budgets
- Testing expectations
- Deployment and CI requirements
- Commit, review, and release processes
- How implementation agents should work from the governing documents

This is the contract between the product vision and the engineering team.

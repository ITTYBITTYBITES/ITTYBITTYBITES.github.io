# CURRENT MODE

**Platform v2 — Product Architecture Phase**

**Date**: 2026-07-07

## Current Directive

**Implementation work is paused.**

The Chief Product Architect is authoring the governing documents in priority order:

1. Product Constitution
2. Platform Blueprint
3. Design System
4. Engineering Standards + supporting strategies

Until the **Product Constitution** (and relevant sections of subsequent documents) are published, **no new creative, visual, navigational, or experience development work** should be performed.

## Allowed Work (only if explicitly requested)

- Bug fixes
- Tooling / DX improvements
- Performance or accessibility maintenance on the existing foundation
- Narrow technical tasks with a clear reference to existing code

## Forbidden (without explicit new Build Order from the architect)

- New experiences
- Visual design or theming changes
- Navigation or information architecture changes
- New pages or surfaces
- Any "make it look better" work

## When Work Resumes

Implementation agents will receive Build Orders in this strict format:

> Implement Section X.Y.Z of [Governing Document] exactly as specified.

All Build Orders will include the standard completion checklist (see `docs/BUILD_ORDERS/README.md`).
